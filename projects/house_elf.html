Challenges

Bad start

When developing personal scripts, I seldom think about scalability. I need something that suits my needs as soon as possible, not something that requires time and extensive planning for future use; scalability comes at the expense of time and effort.

The development is still ongoing, and is unlikely to cease anytime soon. This project taught me JavaScript, Node.js, the necessity of database, why use Docker, merit of using a cloud.

Therefore, this section will be more like a developer's journal.

Discord.py vs. Discord.js

The first decision was 

Cloud Deployment

This decision was straightforward. To run this bot 24/7, I had two options:

1. Run the Python script on my computer 24/7

2. Let a cloud server run my script for me

After some research, I discovered Heroku offers free hosting service for small web applications. This was ideal for my case because the bot would only require a small amount of computational power.
